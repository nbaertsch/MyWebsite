About This Server

The Server

This profile site is served by an nginx server running on a Raspberry Pi 2B. If you want to put a Pi webserver together yourself, all you need is a Rapsberry Pi, a micro SD, a stable internet connection, and an ISP that allows you to port-forward.

Raspberry Pi Setup

Before you can begin doing anything with your Pi, it needs an operating system. Personally, I installed Raspbian. I would reccomend the Lite image if you plan on using your Pi as a webserver. This distro is 'Lite' in that it has no graphical user interface, meaning you will be doing evrything through the command line. Once you've downloaded the image, you'll need to burn it to your micro SD card, which acts as the hard drive of the Pi. I recommend installing Etcher for this.

Now, because this Pi will be headless (i.e. no monitor attached), we need to enable ssh for the image before booting it up on the machine. This is very easy to do. Simply add an empty text file named "ssh" (NOT ssh.txt) into the root of the SD card, and your set to go. Now you can load the card into your Pi, plug it into your router, and turn it on. Raspbian knows how to communicate with your router over the ethernet connection, so your router will assign it an IP address, which you'll need to know if you ever want to do anything with the machine other than look at it. This takes us to the next step...

Port Forwarding

Port forwarding is a technique by which your router passes connections from the outside world to one of the machines on your LAN. Because of the variety of ISP's and routers available, this step is going to be unique to you. The basic steps are as follows:

  1. Login to your router's web-portal and find your Pi as a connected device.
  2. Set your Pi's IP to static, or reserved, and write it down.
  3. Setup forwarding rules to forward incoming port 80 requests to port 80 on the Pi

NginX Instalation and Setup

Once that's all done you can ssh into the machine with the default login credentials (User: pi Password: raspberry). The first thing you'll want to do is update the Pi's repos with sudo apt-get update and then sudo apt-get upgrade. Now that aptitude is updated, installing nginx is just one line of commands sudo apt-get install nginx.

Now that everythings installed, you can run sudo systemctl start nginx to begin serving web pages from the Pi. To verify this, use a web browser to navigate to the IP address of the Pi. You should be greeted with the default nginx page, congrats!
